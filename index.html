Gradients

One approved gradient — Wine to Coral. Use only for decorative backgrounds, hero blocks, and accent shapes. Never apply to text or icon fills.

Usage rules
Do

Use the gradient as a decorative background on hero sections and accent shapes.

Overlay white or light text on the gradient — white achieves 3.2:1 minimum on the Coral end (sufficient for large/bold text).

Keep gradient use limited to 1–2 surfaces per layout.

Don't

Use the gradient for body text backgrounds — contrast is insufficient for paragraph copy.

Apply the gradient to icon fills or UI component fills.

Introduce non-brand colors into the gradient stop.

Dark mode

All RDC HTML output adapts automatically to OS dark mode preference via @media (prefers-color-scheme: dark). Force overrides are available. Brand colors never change.

Force override
<html data-theme="dark">   <!-- force dark mode -->
<html data-theme="light">  <!-- force light mode -->
<html>                      <!-- follow OS preference (default) -->
The FOUC prevention script in <head> reads the saved preference from localStorage and applies the correct data-theme attribute before CSS paints — preventing a flash of wrong theme on load.

Token discipline
Do

Always use CSS tokens: var(--gray-700), var(--white), var(--page-bg).

Dark mode adapts automatically with zero extra code when tokens are used correctly.

Don't

Hardcode hex values for text or background: #444444, #F3F3F3.

Those values won't adapt and will break dark mode rendering.

Layout toggle

A fixed floating control injected by rdc.js that lets readers adjust alignment, text size, and theme without touching the code. Include in every HTML deliverable.

Controls

AlignCenter · LeftToggles content column between centered (default) and left-aligned

Size — Editorial19px / 1.85 lhLong-form reading, wide-format layouts

Preferences are persisted in localStorage under the key rdc-prefs and restored on every subsequent page load.

Usage rules
Do

Include rdc.js before </body> on every generated HTML deliverable — the toggle is injected automatically.

Keep Center and Default as the active starting state on all new pages.

Don't

Modify the toggle's position: fixed or z-index — it is designed to float above all content.

Remove the toggle from client-facing deliverables unless embedding in a context where a fixed element would conflict with an outer shell.

Accessibility

All RDC HTML output targets WCAG 2.1 Level AA. As of v2.1, this library passes axe-core audit with zero violations across 33 checks.

Color contrast — key pairings
PairingRatioNormal textLarge textNotes
White on Squid 16.1:1 Pass Pass Primary dark surface — excellent
Squid on White 16.1:1 Pass Pass Primary body text
White on Wine 10.6:1 Pass Pass Reversed labels on Wine surfaces
Coral on Squid 5.6:1 Pass Pass Active nav, accents on dark bg
Coral on White 3.2:1 Fail Pass Large/bold text and eyebrows only — never body copy
Gray-500 on White 5.5:1 Pass Pass Updated in v2.1 from #888 → #696969
Eyebrow (#C8002A) on White 6.0:1 Pass Pass Accessible coral for small text on light bg
Coral contrast constraint
Coral (#FF4F58) achieves only 3.2:1 against white — sufficient for large headings, button labels, and eyebrow tags (≥18px or ≥14px bold), but not for paragraph copy or small UI labels on light backgrounds. Use #C8002A (the accessible variant) for small eyebrow text.
Interactive elements
ElementRequirement
Focus indicatorsAll focusable elements must have a visible focus ring. Never suppress outline without a custom visible replacement. RDC CSS provides a Coral ring on inputs.
ButtonsAlways include descriptive label text. Icon-only buttons require aria-label.
LinksColor alone is insufficient (WCAG 1.4.1). Pair color with underline or another non-color cue.
Form labelsAssociate every label with its input via matching for/id or aria-label.
ImagesDescriptive alt text on all informational images. alt="" on purely decorative images.
Keyboard accessAll scrollable <pre> blocks receive tabindex="0" automatically from rdc.js.

Brand rules

Core constraints that protect consistency across every touchpoint. When in doubt, ask the design lead before deviating.

Logo
Do

Use approved Box files. Match logo variant to background — light logo on dark, dark logo on light.

Maintain clear space equal to the logo's bounding square on all four sides.

Display the full lockup at ≥85px digital width.

Don't

Recreate or approximate the logo from scratch. Add drop shadows, glows, or effects.

Stretch, skew, rotate, or recolor the mark.

Place the logo below the 85px minimum digital size.

Color
Do

Lead with Coral for all CTAs and primary actions.

Use Squid to anchor dark surfaces. Use the Wine-to-Coral gradient for decorative accents only.

Limit to 3 brand colors per layout.

Don't

Use Mist as a primary page background — it is accent only.

Set Coral body text on white backgrounds (3.2:1 fails WCAG AA for normal text).

Introduce colors outside the approved palette without design lead sign-off.

Typography
Do

Use Suisse Int'l for all licensed brand communications. DM Sans in all HTML.

Sentence case for all headings. Build hierarchy through weight, not size alone.

Ensure body text is accessible: 15px minimum, gray-700 or darker.

Don't

Use more than 3 weights on a single page or spread.

Title case or all-caps for headings (eyebrows excepted).

Mix Suisse Int'l and DM Sans in the same HTML file.
