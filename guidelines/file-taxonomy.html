File taxonomy & naming
RDC's file naming convention — what each segment means, when to use it, and an interactive builder to generate compliant filenames.

Overview

Every file RDC creates follows the same taxonomic naming system. The convention makes files self-describing — you can determine the client, fiscal year, project class, and version state from the filename alone, without opening it.

Why it matters

  • Findability. Files sort predictably in SharePoint and Windows Explorer. MSFT files cluster together; FY26 files cluster together.
  • Archive integrity. When a project ends, you can identify and move every file belonging to it without guessing.
  • Version clarity. The revision suffix (R1, R2, FINAL) eliminates "which version is this?" confusion.
  • Stewardship. Creator and AM initials establish ownership without needing metadata.

The two patterns

There are two filename patterns depending on the file type:

General files
[KINGDOM]-[PHYLUM]-[CLASS]-[GENUS]-[CREATOR]-[SPECIES].[ext]

Use for any regular project file, deliverable, or internal document.

SOW / contract documents
[KINGDOM]-[PHYLUM]-[CLASS]-[AM]-[YYYY-MM-DD]-[CLIENT]-[GENUS]-[RESOURCE]-[SPECIES].[ext]

The contract fields (AM, date, client, resource) are individually optional — include whichever apply to the document.

Naming convention

Each segment of a filename is a rank in the taxonomy, separated by hyphens. Spaces are never permitted within any rank — use CamelCase for multi-word values.

Rank definitions

Rank What it is Format Required?
Kingdom Client prefix or "RDC" for internal files MSFT / RDC / NMSDC Always
Phylum Fiscal year or calendar year FY26 / 2026 Always
Class PO number (client) or functional category (RDC) PO12345 / SOW / Internal Always
AM Initials Account manager or drafter initials BJ / AM / CT SOW / contract
Draft Date ISO date the document was drafted 2026-05-29 SOW / contract
Client Name Individual or team the document is for AdrienneKrefft / FinOpsTeam SOW / contract
Genus What the file actually is — its identity BrandSlides / BudgetModel Always
Creator File owner's initials (general files only) BJ / CT Optional
Resource RDC team member assigned; always wrapped in [ ] [SStL] SOW / optional
Species Version state R1 / DRAFT / FINAL Recommended

Rules to remember

  • No spaces anywhere in the filename. Use CamelCase within a rank segment.
  • Hyphens separate ranks. Do not use underscores.
  • Kingdom is always uppercase (MSFT, not msft).
  • Use FY## for client projects tied to a fiscal year; YYYY for calendar-anchored internal documents.
  • The Resource segment is always wrapped in square brackets: [SStL].
